## 2. 对项目的理解

### 2.1 项目核心目标

**zlr-explore**是一个端到端的多无人机未知环境探索框架：

| 特性 | 说明 |
|------|------|
| **端到端学习** | SAC算法直接从观测映射到离散动作 |
| **多智能体协作** | 3-6架同构无人机协同探索（数量可变）|
| **拓扑图表示** | 40维节点特征（位置2+占据1+访问1+朝向36）|
| **受限视野** | 有限FoV深度相机（如120°）|
| **变尺寸地图** | Swin Transformer处理任意尺寸地图 |
| **隐式会和** | 无人机学会在稀疏通信下自主会和 |

### 2.3 观测空间详细设计

#### 2.3.1 拓扑节点图（40维特征）

**节点生成流程**（FUEL方法，C++转Python）：

```
1. 前沿检测 (Frontier Detection)
   └─> 扫描地图，找到与未知区域相邻的已知自由栅格

2. 区域生长聚类 (Region Growing Clustering) 【核心步骤】
   └─> expandFrontier(): 从种子点开始BFS扩展
   └─> 遍历26邻域，找到连通的前沿栅格
   └─> 形成前沿簇(Frontier Cluster)

3. 计算簇中心 (Cluster Center) 【平均值】
   └─> computeFrontierInfo(): 计算簇内所有栅格位置的平均值
   └─> 该平均值即为节点位置

4. 大簇分割 (可选)
   └─> splitHorizontally(): 如果簇过大，使用PCA找到最佳分割方向
   └─> 沿第一主成分方向将簇分割为两个子簇

5. 降采样
   └─> downsample(): VoxelGrid滤波，减少栅格数量

6. 生成视点 (Viewpoints)
   └─> sampleViewpoints(): 在簇周围采样视点位置
```

**关键修正说明**：
- **聚类方式**：区域生长（BFS扩展），不是DBSCAN/K-means
- **中心计算**：**平均值**，不是PCA计算的中心
- **PCA用途**：仅用于**分割过大的簇**，找到最佳分割方向（第一主成分方向）

**节点特征（40维）**：

| 维度 | 特征 | 说明 | 数据类型 |
|------|------|------|----------|
| 0-1 | 位置 [x, y] | 需要归一化（如除以地图尺寸）| float32 |
| 2 | 占据信息 | 多值分类：0=未被占据, 1=当前无人机占据, 2=其他无人机占据 | float32 |
| 3 | 访问信息 | 0=前沿节点(未访问), 1=已探索 | float32 |
| 4-39 | 朝向信息（36维） | 每10°一个扇区，值为该扇区内可见的未知栅格数量 | float32 |

**边的连通性**：两点连线不经过未知区域或障碍物则连通

**通信机制**：
- 范围限制：一定距离内（如SENSOR_RANGE）才能通信
- 通信内容：共享拓扑节点图、位置信息（取上一次通信的位置）

#### 2.3.2 地图编码器（Swin Transformer）

| 配置项 | 说明 |
|--------|------|
| 输入 | 四值栅格地图（障碍物/未知/已知/前沿）|
| 尺寸处理 | 边界长度 = max(探索范围_x, 探索范围_y) × 1.2（SCALE_FACTOR）【可配置】|
| 尺寸适配 | 如不能整除则在1.2倍基础上缩减到合适尺寸（resize/pad）|
| 融合方式 | 交叉注意力机制将地图编码融入节点特征（K,V=地图特征, Q=节点特征）|
| 输出 | 与节点特征拼接的地图特征 |

**地图编码器处理流程**：
```
1. 计算探索范围: max_range = max(max_x - min_x, max_y - min_y)
2. 计算边界长度: size = max_range × 1.2
3. 提取地图区域: 以当前位置为中心，提取size×size区域
4. 尺寸适配: 如果size不能整除patch_size，则resize到可整除尺寸
   - 方式: 保持1.2倍比例，适当缩小边长至可被patch_size整除
5. Swin编码: 输入适配后的地图，获取特征图
6. 交叉注意力: 将地图特征作为K,V，节点特征作为Q进行融合
```

**parameter.py配置参数**【自由发挥部分】：
```python
MAP_ENCODER_CONFIG = {
    'scale_factor': 1.2,           # 边界扩展系数【可调整：1.0-1.5】
    'patch_size': 4,               # Swin Transformer patch size
    'embed_dim': 128,              # 嵌入维度
    'num_layers': 4,               # 编码器层数
}
```

#### 2.3.3 机器人状态
- 当前无人机的位置 [x, y]（归一化）
- 当前无人机的朝向 θ（0°~360°，归一化到0~1）

### 2.4 动作空间设计

| 属性 | 说明 |
|------|------|
| 类型 | 离散动作空间 |
| 动作 | 选择候选前沿节点 + 选择该节点的最佳朝向 |
| 维度 | 不固定，取决于当前候选前沿节点数量 |

**输出结构**：
1. **节点选择头**：输出每个候选节点的选择概率（Softmax）→ 选择概率最高的节点
2. **朝向预测头**：输出每个候选节点的最佳朝向（Sigmoid输出0~1）→ 映射到0°~360°

**详细说明**：
```
输入: 候选前沿节点列表 [node_1, node_2, ..., node_n]
      ↓
┌─────────────────┐    ┌─────────────────┐
│   节点选择头     │    │   朝向预测头     │
│  (Pointer Net)  │    │  (朝向编码器)    │
│                 │    │                 │
│  输出: n个概率   │    │  输出: n个朝向值 │
│  [p1, p2, ...,  │    │  [o1, o2, ...,  │
│   pn] → Softmax │    │   on] → Sigmoid │
└─────────────────┘    └─────────────────┘
      ↓                        ↓
  选择节点 i              获取朝向 oi
```

**FOV计算示例**：
```
朝向预测值: 0.5
FOV视野: 120° (FOV_ANGLE)
朝向中心线: 0.5 × 360° = 180°
视野范围扇形: [180° - 60°, 180° + 60°] = [120°, 240°]

注：每个前沿节点对应一个最佳朝向预测值
```

### 2.5 奖励函数设计

**整体结构**：`R = w₁R₁ + w₂R₂ + w₃R₃ + w₄R₄ + w₅R₅`

| 奖励项 | 名称 | 计算方法 | 参考权重【自由发挥】 | 参考 |
|--------|------|----------|----------|------|
| R₁ | 航行距离代价 | A*算法计算当前位置到目标点距离 | `W_DISTANCE = -1/512` | 代码1+文章1 |
| R₂ | 朝向代价 | cos(当前朝向 - 目标朝向) | `W_HEADING = 0.1` | 代码1+文章1 |
| R₃ | 局部信息增益 | 下一目标点视线角度下可见的未知栅格数量 | `W_LOCAL_UTIL = 1/50` | - |

#### 2.6.2 Critic网络（Privileged Critic）

采用**中心化训练去中心化执行 (CTDE)** 范式，引入论文2提出的**特权Critic**设计：

**输入特权信息：**
- 全局真实地图（而非单个机器人的局部地图）
- 所有机器人的真实位置与动作
- 完整的全局拓扑图（包含所有机器人共享的节点信息）

**结构流程：**
1. Critic使用与Actor**相同的图编码器和地图编码器结构**
2. 编码器输出所有机器人的增强节点特征后，提取当前机器人 $i$ 的节点特征 $\hat{h}_c^{(i)}$ 与其他机器人 $j$ 的节点特征 $\hat{h}_c^{(j)}$
3. 将其他机器人的状态-动作对与当前机器人的query特征拼接，输入**注意力层**进行信用分配（参考论文2的Actor-Attention-Critic机制）
4. 最后通过**2层前馈网络 (Feed-Forward)** 映射到标量Q值：
   $$Q_{\phi}(s_t, a_t) = \text{FFN}([\hat{h}_c^{(i)} \parallel \{\hat{h}_c^{(j)}, a_t^{(j)}\}_{j \neq i}])$$

#### 2.6.3 解码器（双头输出）

```
- DecoderLayer (embedding_dim=128, n_head=4, n_layer=1)【自由发挥】
- 当前状态嵌入: Linear(embedding_dim * 2, embedding_dim)
- 指针网络: SingleHeadAttention (用于节点选择)
- 朝向预测头: Linear(embedding_dim, 1) + Sigmoid 输出0~1
```

#### 2.8.2 地图格式

**原始地图（PNG灰度图像，8-bit）：**

| 像素值 | 含义 |
|--------|------|
| 208 | 机器人初始出生位置（仅在原始地图中存在）|
| >150 | 障碍物区域（墙壁等）|
| 50-80 | 障碍物区域（其他灰度障碍物）|
| 其他 | 空闲可通行区域 |

**处理后地图（二值化转换）：**

| 像素值 | 含义 |
|--------|------|
| 255 (FREE) | 空闲可通行区域 |
| 1 (OCCUPIED) | 障碍物区域 |
| 127 (UNKNOWN) | 未知未探索区域（机器人belief初始化值）|
| 255 | 前沿区域（保持自由栅格值，不需要额外设置）|

**处理流程：**
1. 读取PNG图像 → 2倍下采样（`block_reduce`）→ 二值化
2. 二值化规则：`(像素>150) | (50≤像素≤80)`为障碍物，其余为空闲
3. 最终转换：`ground_truth = 二值化结果 × 254 + 1`（得到255或1）
4. 坐标系统：栅格坐标（图像坐标系，y向下），世界坐标以初始位置为原点，`CELL_SIZE=0.4`米/栅格
5. 机器人位置独立存储在`robot_locations`数组中，包含世界坐标(x,y)和航向角(0-360°)

#### 2.8.3 评价指标

| 指标 | 说明 |
|------|------|
| 探索覆盖率 | 已探索区域 / 总可探索区域 |
| 探索成功率 | 是否完成探索（如覆盖率>95%）|
| 完成时间 | 完成探索所需的步数/时间 |
| 路径平均长度 | 所有机器人航行距离的平均 |
| 机器人间重复探索比例 | 重叠区域的比例 |


### 参数配置规范

1. **奖励函数权重** - `REWARD_WEIGHTS` 字典，支持独立调整

3. **环境参数** - 传感器范围、FOV角度、机器人数量等
4. **训练参数** - 学习率、批次大小、课程学习阶段等
5. **地图编码器参数** - scale因子、patch大小等

**编码规范**：禁止在代码中硬编码任何数值，统一从parameter.py导入
