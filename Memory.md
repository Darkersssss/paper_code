## 🔒 执行流程规范

根据 `Project.md` 的强制要求，**每个 Step 必须严格遵循以下流程**：

```
┌─────────────────────────────────────────────────────────────┐
│  Step 执行标准流程（缺一不可）                                │
├─────────────────────────────────────────────────────────────┤
│  1. AI 明确说明：接下来要干什么 + 涉及文件 + 预期结果        │
│  2. 用户回复"确认"或"同意"                                  │
│  3. AI 执行任务清单中的内容                                  │
│  4. 编写测试代码到 test.py，运行测试                         │
│  5. 判断：所有测试项通过？                                   │
│     ├─ 是 → 更新 Memory.md 状态为"✅ 已完成"，进入下一步    │
│     └─ 否 → 修复问题，重试（重试次数+1），回到步骤 4        │
└─────────────────────────────────────────────────────────────┘
```

**关键规则：**
- **测试通过是模块完成的唯一标准**。未通过测试的 Step 不得标记为完成。
- 每次执行前必须获得用户确认，禁止擅自执行。
- 所有状态变更、报错信息、测试结果必须实时记录到本文件中。
